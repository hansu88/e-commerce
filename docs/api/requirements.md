# 요구사항 문서

## 1. 기능적 요구사항

### 상품 관리
- 판매 중인 상품 목록 조회
- 상품 상세 조회 (색상, 사이즈, 가격, 재고)
- 인기 상품 통계 (최근 3일, Top 5)

### 장바구니
- 장바구니 담기, 조회, 삭제
- 담긴 상품 총 금액 확인
- 재고 실시간 확인

### 주문/결제
- 장바구니 기반 주문 생성
- 잔액 또는 결제 수단으로 결제
- 재고 차감 및 주문 상태 변경
- 쿠폰 적용 가능

### 쿠폰
- 선착순 발급 (한정 수량)
- 쿠폰 유효성 검증
- 사용 이력 관리

### 데이터 연동
- 주문 데이터 외부 전송
- 실패 시에도 주문 정상 처리

---

## 2. 비기능적 요구사항
- API 응답 시간 500ms 이하
- 트랜잭션 처리 시 일관성 보장 (ACID)
- 동시성 처리 고려 (동시 주문, 쿠폰 선착순)
- 로그 기록 및 장애 추적 가능
- H2 및 추후 MySQL 호환

---

## 3. 사용자 역할 및 권한
- 일반 사용자: 상품 조회, 장바구니, 주문, 쿠폰 조회/사용
- 관리자: 상품 등록/수정/삭제, 쿠폰 발급 관리
- 시스템: 재고 차감, 주문 데이터 외부 연동
